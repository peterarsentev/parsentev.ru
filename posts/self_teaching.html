Как стать Java программистом? Пошаговый план.

Ноябрь 29, 2015 | Автор: Арсентьев Петр

В этой статье я рассказываю, как стать Java программистом. Здесь изложена пошаговая стратегия. С чего следует начинать. Какие книги нужно прочитать обязательно, какие технологии и библиотеки нужно изучить, по окончанию вашей подготовки предлагаю выполнить тестовое задание. Тестовое задание покрывает все технологии необходимые для устройства на работу Java программистом.

Прежде чем разбираться, как стать Java программистом давайте с начало осознаем зачем мы хотим стать программистом и именно Java программистом.
Истинным мотивом приобретения необходимых навыков в любой профессии должен быть подлинный интерес к этому виду деятельности. То есть вам действительно должно нравится писать код, придумывать алгоритмы, заниматься оптимизацией бизнес процессов и решать многие другие задачи, с которыми сталкивается программист.
Если вашем мотивом являются только финансовая сторона, а Java программисты зарабатывают хорошие деньги, то хочу сразу вас разочаровать – больших результатов, а тем более удовольствия от проделанной работы вы не получите. Даже больше, вам будет скучно и уныло.
Поэтому сперва ответьте на простой вопрос какие мотивы лежат в основе вашего желания стать Java программистом?
В начале любого пути нужно определить откуда мы двигаемся и куда ходим прийти в конечном итоге.
То есть весь маршрут достижения данной цели можно отобразить на простой прямой из двух точек А и Б.
Точка А описывает ваше текущее состояния. Ниже приведен список вопросов для определения вашей начальной точки старта.

Ответить на вопросы.
1. Может ли Вы установить ОС Window?
2. Может ли Вы установить ОС Linux?
3. Можете ли Вы работать в командной стороке?
4. Знаете ли Вы что такое базы данных? Каким образов храняться данные в базе данных?
5. Знаете ли Вы что такое языки программирования?
6. Имеете ли Вы опыт программирования с использованием любого языков программирования?
7. Знаете ли Вы что такое алгоритмы? Можете привести пример алгоритма?
8. Сможете написать алгоритм для прохождения статического лабиринта? Движения задается в шагах и направлению. Например: 2 шага вперед, 3 шага влево, 8 шагов прямо. Выход.
9. Знаете ли Вы как работает браузер?
10. Что такое беспроводная сеть? Приведите примеры.

Если на большинство вопросов вы ответили положительно, значит у вас есть все шансы стать Java программистом. К сожалению, если большинство ответов отрицательных – то вам нужно начинать с основ. Не расстраивайтесь, эти знания можно легко изучить самостоятельно.

Теперь перейдем ко второй точке нашего пути.
Для того что бы составить список необходимых навыков, что бы достичь данной точки нам нужно:
1. Собрать список всех вакансий, которые доступны в вашем городе.
2. Выписать все требования, указанные в каждой вакансии.
3. Сгруппировать это список.

Полученный список и будем вашей второй точкой.

При выборе конечного назначения, обычно руководствуются минимальными возможностями. Например, с работой программиста выбирают устройство на позиции junior. Следовательно, список будет состоять из требований к позиции junior java developer. И здесь есть хитрый психологически момент. В таком случае выполнение задачи на 100% - это получить все навыки из вашего списка, но обычно что бы сделать задачу реально на 100% нужно приложить очень много усилий. Всегда возникают какие-нибудь мелкие недочеты, правки, переделки и желания усовершенствовать. Поэтому я рекомендую ставить цель на 30% выше. То есть готовиться как будто вы хотите занять позицию middle java developer.

Ниже приведен минимальный список технологий и библиотек необходимых для устройства на позиции Java Junior developer.
1. Понятия наследования, инкапсуляции и полиморфизма. (Interface. Abstract Class, Inner Class and so on).
2. SOLID principles
3. Collections: List, Set, Tree, Hashable.
4. JDBC
5. Multithreading (synchronize, monitor, wait, notify, notifyAll, deadlock).
6. Servlet, JSP, JSTL.
7. SQL.
8. Junit.
9. Основы HTML, CSS, JS.
10. Ajax.
11. Maven.
12. GIT.
13. Tomcat.

Так же ниже я привел список литературы, который необходимо прочитать любому Java программисту.

1. Head first Java.
2. Head first Servlet, JSP.
3. Head first SQL.
4. Head first Design Patterns.
5. Effective Java.
6. Официальную документацию по Tomcat Git Maven.
После того как вы решили, что вы уже готовы начать искать работу я предлагаю вам сперва выполнить тестовое задания. Что бы вы могли проверить свои знания на практике.

Тестовое задание.

Необходимо создать веб приложение – система заявок. Заявки должны иметь древовидную структуру неограниченной вложенности. Заявка должна хранить информацию: Имя. Описание, Дата создания, Автор. К заявке можно загрузить файлы. И добавить комментарии. Комментарии должны содержать информации об авторе и дате создания.
В системе должны быть два типа пользователей модераторы и обычные пользователи. Редактировать заявку может автор либо модератор. Добавлять комментарии могут все. Редактировать коммендации нельзя.
В системе нужно входить по логину и паролю. Заводить новых пользователей может только модератор.
Пользовательский интерфейс должен быть представлен одностраничным сайтом. Страница должна быть разделена на две части. В левой стороне дерево всех задач, а в правой текущий список. Предусмотреть два фильтра. Плоский – показывает только заявки и на одном уровне и вложенный. Отображает все заявки включая вложенные.
Для пользовательского интерфейса использовать Angular JS.
Все данные должны храниться в базе данных – PostgreSQL.

Заключение.

Как видите вам предстоит пройти не легкий путь. Но идущий пройдет дорогу. Так что начинайте смело. Как и в любой новом пути легче всего его пройти, если вас по нему проведет опытный проводник. Наиболее быстрый способ овладеть данными знаниями пройти курс под моим руководством.
